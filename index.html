  • git init

git init komutu, mevcut bir dizini bo bir Git deposu olarak balatmak iin kullanlan bir Git komutudur. Bu komutu kullanarak, proje dosyalarnz Git deposuna ekleyebilir ve Git'in versiyon kontroln balatabilirsiniz.

git init komutunu kullanmann temel admlar u ekildedir:

  1. Git deposunu oluturmak istediiniz dizine gidin. rnein, projenizi oluturmak istediiniz dizini an veya cd komutunu kullanarak o dizine gein.
  2. Ardndan, aadaki komutu girin:

>>> git init                                 

Bu komut, mevcut dizini bo bir Git deposu olarak balatacaktr. Dizine .git adnda bir klasr oluturulacak ve bu klasrde Git'in tm depo verileri saklanacaktr.

rnek bir senaryo olarak, "my_project" adnda bir dizinde yeni bir Git deposu oluturmak istediinizi varsayalm:

  1. Terminali an ve "my_project" dizinine gein:

>>> cd my_project                        

  1. Ardndan, git init komutunu kullanarak Git deposunu balatn:

>>> git init                                

Bu komutu altrdktan sonra, "my_project" dizininde .git adnda gizli bir klasr oluturulacak ve Git deposu oluturulmu olacak.

rnek olarak aadaki gibi bir terminal kts grebilirsiniz:

>>> Initialized empty Git repository in /path/to/my_project/.git/

git clone, uzaktaki bir Git deposunu yerel bir dizine kopyalamak iin kullanlan bir Git komutudur. Bu komut, projeyi tamamyla almanz salar ve projenin tm gemiini ve dosyalarn ieren bir kopya oluturur.

git clone komutu nasl kullanlr?

  1. Terminali an ve kopyalamak istediiniz projenin yerel dizinine gein. rnein, projeyi my_project adnda bir dizine kopyalamak istediinizi varsayalm:

>>> cd my_project                                

  1. git clone komutunu kullanarak projeyi kopyalayn. Bunun iin, projenin URL'sini kullanmanz gerekmektedir. rnein, GitHub'da yer alan bir projeyi kopyalamak iin aadaki komutu kullanabilirsiniz:

>>> git clone <repo_url>                        

<repo_url> yerine, kopyalamak istediiniz Git deposunun URL'sini eklemeniz gerekmektedir. Bu URL, projenin bulunduu Git servisi tarafndan salanr.

rnein, bir GitHub reposunu kopyalamak iin u komutu kullanabilirsiniz:

>>> git clone <repo_url>                        

Bu komutu altrdktan sonra, Git, uzaktaki projeyi klonlayacak ve projenin tam bir kopyasn yerel dizine oluturacaktr.

git add komutu, alma dizinindeki dosyalar Git deposuna eklemek iin kullanlan bir Git komutudur. Bu komut, dosyalarn Git tarafndan izlenmesini salar ve bir sonraki   git commit ilemine dahil edilmelerini salar. (git commit ifadesi hemen bir sonraki balkta aklanmtr.)

git add komutunu temel seviyede aklamak iin aadaki admlar takip edebilirsiniz:

  1. Terminali an ve Git deposunun bulunduu dizine gein. rnein, my_project adnda bir Git deposuna sahip olduunuzu varsayalm:

>>> cd my_project                                

  1. git add komutunu kullanarak dosyalar Git deposuna ekleyin. Tm dosyalar eklemek iin . (nokta) karakterini kullanabilirsiniz:

>>> git add .                                        

veya belirli bir dosyay eklemek iin dosya adn belirtebilirsiniz:

>>> git add .        <dosya_adi.ext>                

Bu komutu altrdktan sonra, Git, belirtilen dosyalar depoya ekleyecektir.

rnein, my_project deposunda index.html ve style.css adnda iki dosyanz olduunu varsayalm. Bu dosyalar Git deposuna eklemek iin aadaki komutu kullanabilirsiniz:

>>> git add index.html style.css                

Bu komutu altrdktan sonra, Git dosyalar depoya ekleyecek ve bir sonraki  git commit ilemine dahil edilecektir.

git commit komutu, dosya deiikliklerini Git deposunda kalc olarak kaydetmek iin kullanlan bir Git komutudur. Bu komut, bir dizi deiiklii bir paket halinde oluturur ve bir mesajla birlikte depoya kaydeder.

git commit komutu nasl kullanlr?

1. Terminali an ve Git deposunun bulunduu dizine gein. rnein, my_project adnda bir Git deposuna sahip olduunuzu varsayalm:

>>> cd my_project                                                                

  1. git commit komutunu kullanarak deiiklikleri kaydedin. Aadaki komutu kullanabilirsiniz:

-m seenei, bir commit mesaj belirtmenizi salar. Commit mesaj yerine, kaydettiiniz deiiklikleri aklayan bir mesaj eklemeniz gerekmektedir.

rnein, index.html dosyasnda yaplan deiiklikleri kaydetmek iin aadaki komutu kullanabilirsiniz:

Bu komutu altrdktan sonra, Git, yaplan deiiklikleri bir commit olarak kaydedecek ve bir commit kimlii (commit ID) retecektir.

Commit mesaj, yaplan deiikliklerin zeti ve nedeni hakknda aklayc bir ekilde olmaldr. Dier gelitiriciler veya sizin gibi projenin iinde yer alan kiiler, commit mesajlar araclyla yaplan deiiklikleri anlayabilir ve takip edebilir.

  • git push

git push, yerel Git deposundaki yerel deiiklikleri bir uzak sunucuya gndermek iin kullanlan bir Git komutudur. Bu komut, yerelde yaptnz commit'leri ve deiiklikleri uzaktaki bir depoya yklemenizi salar.

git push komutu nasl kullanlr?

  1. Terminali an ve alma dizinine gein. alma dizini, Git deposunun bulunduu dizini temsil eder.

>>> cd my_project                                

  1. git push komutunu kullanarak deiiklikleri gnderin. Genel olarak, git push komutunu aadaki ekilde kullanabilirsiniz:

>>> git push        <repo> <branch>                 

  1.  <repo>, deiikliklerin gnderilecei uzak sunucuyu temsil eder. Bu genellikle origin olarak adlandrlan varsaylan uzak sunucudur.

  1. <branch>, gnderilecek deiikliklerin bulunduu yerel dal temsil eder.
    rnein, yerelde yaptnz deiiklikleri origin adl uzak sunucuya master dalna gndermek isterseniz, aadaki komutu kullanabilirsiniz:

                

>>> git push        origin master                

  1. Komutu altrdktan sonra, Git, yerel deiiklikleri belirtilen uzak sunucuya gnderecektir. Uzak sunucudaki ilgili dal gncellenecek ve deiiklikler orada grntlenebilecektir.

Bu ekilde, git push komutunu kullanarak yerel deiiklikleri uzak bir depoya gnderebilirsiniz.

  • git pull

git pull, uzaktaki bir Git sunucusundan gncellemeleri almak ve yerel depoya entegre etmek iin kullanlan bir Git komutudur. Bu komut, uzak sunucudaki deiiklikleri indirir ve otomatik olarak yereldeki mevcut alma kopyasna birletirir.

git pull komutu nasl kullanlr?

  1. Terminali an ve alma dizinine gein. alma dizini, Git deposunun bulunduu dizini temsil eder.

  1. git pull komutunu kullanarak gncellemeleri aln. Genel olarak, git pull komutunu aadaki ekilde kullanabilirsiniz:

>>> git pull <repo> <branch>                 


         <repo>, gncellemelerin alnaca uzak sunucuyu temsil eder. Bu genellikle origin olarak adlandrlan varsaylan uzak sunucudur. <branch>, gncellemelerin bulunduu uzak dal temsil eder.
rnein, origin adl uzak sunucudaki master dalndaki gncellemeleri almak isterseniz, aadaki komutu kullanabilirsiniz:

git pull origin master

Komutu altrdktan sonra, Git, uzak sunucudaki gncellemeleri indirecek ve yereldeki mevcut alma kopyasna birletirecektir. Uygun birletirme ilemi gerekletirildikten sonra, yereldeki alma kopyanz gncellenmi olacaktr.

Bu ekilde, git pull komutunu kullanarak uzaktaki gncellemeleri alabilir ve yerel depoya entegre edebilirsiniz. Bu, yereldeki alma kopyanz gncel tutmanz ve dier ekip yeleriyle ortak bir kod tabannda almasn salar.

git branch, Git deposundaki mevcut dallar listelemek, yeni bir dal oluturmak veya mevcut bir dala gemek iin kullanlan bir Git komutudur. Dallar, Git'in paralel ve bamsz alma kopyalar oluturmanz salar.

git branch komutu nasl kullanlr?

  1. Terminali an ve alma dizinine gein. alma dizini, Git deposunun bulunduu dizini temsil eder.
  2. git branch komutunu kullanarak mevcut dallar listele. Sadece git branch komutunu altrdnzda, mevcut olan tm dallarn bir listesi grntlenecektir. Aktif dal, komut ktsnda yldz (*) iaretiyle gsterilir.

        

>>> git branch                                

rnein, aadaki ktda "master" ve "develop" adnda iki dal olduunu grebilirsiniz:

>>> * master develop                        

  1. Yeni bir dal oluturmak iin git branch komutunu ve yeni daln adn kullann:

>>> git branch <yeni-dal-adi>                

rnein, "feature-branch" adnda yeni bir dal oluturmak iin u komutu kullanabilirsiniz:

>>> git branch feature-branch                

  1. Mevcut bir dala gemek iin git checkout komutunu kullanabilirsiniz:

>>> git checkout <new-branch>                


rnein, "develop" dalna gemek iin u komutu kullanabilirsiniz:

>>> git develop <new-branch>

git checkout, Git deposunda farkl dallar arasnda gei yapmak, belirli bir commit'e geri dnmek veya yeni bir dal oluturmak iin kullanlan bir Git komutudur. Bu komut, alma kopyasn ve indeksi belirli bir duruma getirmenizi salar.

  1. Terminali an ve alma dizinine gein. alma dizini, Git deposunun bulunduu dizini temsil eder.
  2. Dallar arasnda gei yapmak iin git checkout komutunu ve hedef daln adn kullann:

>>> git checkout <branch>                


Bu komut, alma kopyasn ve indeksi belirtilen dala gre gnceller. Var olan deiikliklerinizin olmamas veya commitlenmemi deiikliklerin dal deitirmeye engel olmamas durumunda gei sorunsuz olacaktr.

  1. Belirli bir commit'e geri dnmek iin git checkout komutunu ve hedef commit in kimliini (commit ID veya dallar, etiketleri kullanarak referans alabilirsiniz) kullann:

>>> git checkout <branch>                

Bu komut, alma kopyasn ve indeksi belirtilen commit durumuna geri getirir. Bylece, projenin gemiteki bir durumuna gei yapabilirsiniz. Bu ilem, mevcut deiikliklerinizi kaydedebileceiniz iin dikkatli olmanz gereken bir ilemdir.

  1. Yeni bir dal oluturmak iin git checkout komutunu ve -b bayran kullanarak yeni daln adn belirtin:

>>> git checkout -b <branch_name>

git merge, farkl dallardaki deiiklikleri birletirmek iin kullanlan bir Git komutudur.  Bu komut, bir daldaki deiiklikleri baka bir dala entegre etmenizi salar.

git merge komutu nasl kullanlr?

  2. Birletirmek istediiniz hedef dala gein:

>>> git checkout <branch>                

  1. Hedef daln zerine birletirmek istediiniz kaynak dal belirtin ve git merge komutunu kullann:

>>> git merge <branch>

  1. akma durumunda, Git size akan dosyalar gsterecektir. Dosyalar dzenleyerek veya akmay zerek deiiklikleri birletirme ilemini tamamlayabilirsiniz. Daha sonra, deiiklikleri commitleyerek birletirme ilemini kesinletirebilirsiniz.

Bu ekilde, git merge komutunu kullanarak farkl dallardaki deiiklikleri birletirebilirsiniz. Bu komut, ezamanl olarak farkl zellikleri veya dzeltmeleri gelitirirken ve farkl dallarda alrken proje ynetiminde yaygn olarak kullanlr.
