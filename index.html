EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git init

“git init” komutu, mevcut bir dizini boş bir Git deposu olarak başlatmak için kullanılan bir Git komutudur. Bu komutu kullanarak, proje dosyalarınızı Git deposuna ekleyebilir ve Git'in versiyon kontrolünü başlatabilirsiniz.

“git init” komutunu kullanmanın temel adımları şu şekildedir:

  1. Git deposunu oluşturmak istediğiniz dizine gidin. Örneğin, projenizi oluşturmak istediğiniz dizini açın veya cd komutunu kullanarak o dizine geçin.
  2. Ardından, aşağıdaki komutu girin:

>>> git init                                 

Bu komut, mevcut dizini boş bir Git deposu olarak başlatacaktır. Dizine “.git” adında bir klasör oluşturulacak ve bu klasörde Git'in tüm depo verileri saklanacaktır.

Örnek bir senaryo olarak, "my_project" adında bir dizinde yeni bir Git deposu oluşturmak istediğinizi varsayalım:

  1. Terminali açın ve "my_project" dizinine geçin:

>>> cd my_project                        

  1. Ardından, “git init” komutunu kullanarak Git deposunu başlatın:

>>> git init                                

Bu komutu çalıştırdıktan sonra, "my_project" dizininde “.git” adında gizli bir klasör oluşturulacak ve Git deposu oluşturulmuş olacak.

Örnek olarak aşağıdaki gibi bir terminal çıktısı görebilirsiniz:

>>> Initialized empty Git repository in /path/to/my_project/.git/        

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git clone

git clone, uzaktaki bir Git deposunu yerel bir dizine kopyalamak için kullanılan bir Git komutudur. Bu komut, projeyi tamamıyla almanızı sağlar ve projenin tüm geçmişini ve dosyalarını içeren bir kopya oluşturur.

git clone komutu nasıl kullanılır?

  1. Terminali açın ve kopyalamak istediğiniz projenin yerel dizinine geçin. Örneğin, projeyi “my_project” adında bir dizine kopyalamak istediğinizi varsayalım:

>>> cd my_project                                

  1. “git clone” komutunu kullanarak projeyi kopyalayın. Bunun için, projenin URL'sini kullanmanız gerekmektedir. Örneğin, GitHub'da yer alan bir projeyi kopyalamak için aşağıdaki komutu kullanabilirsiniz:

>>> git clone <repo_url>                        

“<repo_url>” yerine, kopyalamak istediğiniz Git deposunun URL'sini eklemeniz gerekmektedir. Bu URL, projenin bulunduğu Git servisi tarafından sağlanır.

Örneğin, bir GitHub reposunu kopyalamak için şu komutu kullanabilirsiniz:

>>> git clone <repo_url>                        

Bu komutu çalıştırdıktan sonra, Git, uzaktaki projeyi klonlayacak ve projenin tam bir kopyasını yerel dizine oluşturacaktır.

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git add

“git add” komutu, çalışma dizinindeki dosyaları Git deposuna eklemek için kullanılan bir Git komutudur. Bu komut, dosyaların Git tarafından izlenmesini sağlar ve bir sonraki  “ git commit “ işlemine dahil edilmelerini sağlar. (“git commit” ifadesi hemen bir sonraki başlıkta açıklanmıştır.)

“git add“ komutunu temel seviyede açıklamak için aşağıdaki adımları takip edebilirsiniz:

  1. Terminali açın ve Git deposunun bulunduğu dizine geçin. Örneğin, “my_project” adında bir Git deposuna sahip olduğunuzu varsayalım:

>>> cd my_project                                

  1. “git add” komutunu kullanarak dosyaları Git deposuna ekleyin. Tüm dosyaları eklemek için . (nokta) karakterini kullanabilirsiniz:

>>> git add .                                        

veya belirli bir dosyayı eklemek için dosya adını belirtebilirsiniz:

>>> git add .        <dosya_adi.ext>                

Bu komutu çalıştırdıktan sonra, Git, belirtilen dosyaları depoya ekleyecektir.

Örneğin, “my_project” deposunda “index.html” ve “style.css” adında iki dosyanız olduğunu varsayalım. Bu dosyaları Git deposuna eklemek için aşağıdaki komutu kullanabilirsiniz:

>>> git add index.html style.css                

Bu komutu çalıştırdıktan sonra, Git dosyaları depoya ekleyecek ve bir sonraki  “git commit” işlemine dahil edilecektir.

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git commit

“git commit” komutu, dosya değişikliklerini Git deposunda kalıcı olarak kaydetmek için kullanılan bir Git komutudur. Bu komut, bir dizi değişikliği bir paket halinde oluşturur ve bir mesajla birlikte depoya kaydeder.

“git commit” komutu nasıl kullanılır?

1. Terminali açın ve Git deposunun bulunduğu dizine geçin. Örneğin,” my_project” adında bir Git deposuna sahip olduğunuzu varsayalım:

>>> cd my_project                                                                

  1. “git commit” komutunu kullanarak değişiklikleri kaydedin. Aşağıdaki komutu kullanabilirsiniz:

                >>> git commit -m “Commit Başlığı”                                        

“-m” seçeneği, bir commit mesajı belirtmenizi sağlar. ”Commit mesajı” yerine, kaydettiğiniz değişiklikleri açıklayan bir mesaj eklemeniz gerekmektedir.

Örneğin, “index.html” dosyasında yapılan değişiklikleri kaydetmek için aşağıdaki komutu kullanabilirsiniz:

>>> git commit -m "index.html dosyasında değişiklikler yapıldı"        

Bu komutu çalıştırdıktan sonra, Git, yapılan değişiklikleri bir commit olarak kaydedecek ve bir commit kimliği (commit ID) üretecektir.

Commit mesajı, yapılan değişikliklerin özeti ve nedeni hakkında açıklayıcı bir şekilde olmalıdır. Diğer geliştiriciler veya sizin gibi projenin içinde yer alan kişiler, commit mesajları aracılığıyla yapılan değişiklikleri anlayabilir ve takip edebilir.

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git status

“git status” komutu, çalışma dizinindeki dosyaların ve Git deposunun durumunu görüntülemek için kullanılan bir Git komutudur. Bu komut, hangi dosyaların değiştirildiğini, hangilerinin Git tarafından izlendiğini veya izlenmediğini, hangi dosyaların commit edilmeyi beklediğini vb. bilgileri sağlar.

“git status” komutu nasıl kullanılır?

  1. Terminali açın ve Git deposunun bulunduğu dizine geçin. Örneğin, “my_project” adında bir Git deposuna sahip olduğunuzu varsayalım:

>>> cd my_project                                

  1. “git status” komutunu kullanarak dosya durumunu görüntüleyin:

>>> git status                                 

Bu komutu çalıştırdıktan sonra, Git, çalışma dizinindeki dosyaların durumunu gösterecektir.

Örneğin, projenizde yapılan değişiklikleri görmek istediğinizi varsayalım. “git status” komutunu kullanarak, hangi dosyaların değiştirildiğini ve hangilerinin commit edilmeyi beklediğini görüntüleyebilirsiniz.

Örnek çıktı şu şekilde olabilir:

On branch main Your branch is up to date with 'origin/main'. Changes not                

                                                                                

 staged for commit: (use "git add <file>" to update what will be committed)                

                                                                                

(use "git restore <file>" to discard changes in working directory) modified:                 

                                                                                

index.html modified: style.css no changes added to commit (use "git add"                 

                                                                                

and/or "git commit -a")                                                                

Bu çıktıda, “index.html” ve “style.css” dosyalarının değiştirildiği ve henüz commit edilmediği görülmektedir.

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git push

git push, yerel Git deposundaki yerel değişiklikleri bir uzak sunucuya göndermek için kullanılan bir Git komutudur. Bu komut, yerelde yaptığınız commit'leri ve değişiklikleri uzaktaki bir depoya yüklemenizi sağlar.

git push komutu nasıl kullanılır?

  1. Terminali açın ve çalışma dizinine geçin. Çalışma dizini, Git deposunun bulunduğu dizini temsil eder.

>>> cd my_project                                

  1. git push komutunu kullanarak değişiklikleri gönderin. Genel olarak, git push komutunu aşağıdaki şekilde kullanabilirsiniz:

>>> git push        <repo> <branch>                 

  1.  <repo>, değişikliklerin gönderileceği uzak sunucuyu temsil eder. Bu genellikle origin olarak adlandırılan varsayılan uzak sunucudur.

  1. <branch>, gönderilecek değişikliklerin bulunduğu yerel dalı temsil eder.
    Örneğin, yerelde yaptığınız değişiklikleri origin adlı uzak sunucuya master dalına göndermek isterseniz, aşağıdaki komutu kullanabilirsiniz:

                

>>> git push        origin master                

  1. Komutu çalıştırdıktan sonra, Git, yerel değişiklikleri belirtilen uzak sunucuya gönderecektir. Uzak sunucudaki ilgili dal güncellenecek ve değişiklikler orada görüntülenebilecektir.

Bu şekilde, git push komutunu kullanarak yerel değişiklikleri uzak bir depoya gönderebilirsiniz.

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git pull

git pull, uzaktaki bir Git sunucusundan güncellemeleri almak ve yerel depoya entegre etmek için kullanılan bir Git komutudur. Bu komut, uzak sunucudaki değişiklikleri indirir ve otomatik olarak yereldeki mevcut çalışma kopyasına birleştirir.

git pull komutu nasıl kullanılır?

  1. Terminali açın ve çalışma dizinine geçin. Çalışma dizini, Git deposunun bulunduğu dizini temsil eder.

  1. git pull komutunu kullanarak güncellemeleri alın. Genel olarak, git pull komutunu aşağıdaki şekilde kullanabilirsiniz:

>>> git pull <repo> <branch>                 


         <repo>, güncellemelerin alınacağı uzak sunucuyu temsil eder. Bu genellikle origin olarak adlandırılan varsayılan uzak sunucudur. <branch>, güncellemelerin bulunduğu uzak dalı temsil eder.
Örneğin, origin adlı uzak sunucudaki master dalındaki güncellemeleri almak isterseniz, aşağıdaki komutu kullanabilirsiniz:

git pull origin master

Komutu çalıştırdıktan sonra, Git, uzak sunucudaki güncellemeleri indirecek ve yereldeki mevcut çalışma kopyasına birleştirecektir. Uygun birleştirme işlemi gerçekleştirildikten sonra, yereldeki çalışma kopyanız güncellenmiş olacaktır.

Bu şekilde, git pull komutunu kullanarak uzaktaki güncellemeleri alabilir ve yerel depoya entegre edebilirsiniz. Bu, yereldeki çalışma kopyanızı güncel tutmanızı ve diğer ekip üyeleriyle ortak bir kod tabanında çalışmasını sağlar.

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git branch

git branch, Git deposundaki mevcut dalları listelemek, yeni bir dal oluşturmak veya mevcut bir dala geçmek için kullanılan bir Git komutudur. Dallar, Git'in paralel ve bağımsız çalışma kopyaları oluşturmanızı sağlar.

git branch komutu nasıl kullanılır?

  1. Terminali açın ve çalışma dizinine geçin. Çalışma dizini, Git deposunun bulunduğu dizini temsil eder.
  2. git branch komutunu kullanarak mevcut dalları listele. Sadece git branch komutunu çalıştırdığınızda, mevcut olan tüm dalların bir listesi görüntülenecektir. Aktif dal, komut çıktısında yıldız (*) işaretiyle gösterilir.

        

>>> git branch                                

Örneğin, aşağıdaki çıktıda "master" ve "develop" adında iki dal olduğunu görebilirsiniz:

>>> * master develop                        

  1. Yeni bir dal oluşturmak için git branch komutunu ve yeni dalın adını kullanın:

>>> git branch <yeni-dal-adi>                

Örneğin, "feature-branch" adında yeni bir dal oluşturmak için şu komutu kullanabilirsiniz:

>>> git branch feature-branch                

  1. Mevcut bir dala geçmek için git checkout komutunu kullanabilirsiniz:

>>> git checkout <new-branch>                


Örneğin, "develop" dalına geçmek için şu komutu kullanabilirsiniz:

>>> git develop <new-branch>                

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git checkout

git checkout, Git deposunda farklı dallar arasında geçiş yapmak, belirli bir commit'e geri dönmek veya yeni bir dal oluşturmak için kullanılan bir Git komutudur. Bu komut, çalışma kopyasını ve indeksi belirli bir duruma getirmenizi sağlar.

git checkout komutu nasıl kullanılır?

  1. Terminali açın ve çalışma dizinine geçin. Çalışma dizini, Git deposunun bulunduğu dizini temsil eder.
  2. Dallar arasında geçiş yapmak için git checkout komutunu ve hedef dalın adını kullanın:

>>> git checkout <branch>                


Bu komut, çalışma kopyasını ve indeksi belirtilen dala göre günceller. Var olan değişikliklerinizin olmaması veya commitlenmemiş değişikliklerin dal değiştirmeye engel olmaması durumunda geçiş sorunsuz olacaktır.

  1. Belirli bir commit'e geri dönmek için git checkout komutunu ve hedef commit in kimliğini (commit ID veya dalları, etiketleri kullanarak referans alabilirsiniz) kullanın:

>>> git checkout <branch>                

Bu komut, çalışma kopyasını ve indeksi belirtilen commit durumuna geri getirir. Böylece, projenin geçmişteki bir durumuna geçiş yapabilirsiniz. Bu işlem, mevcut değişikliklerinizi kaydedebileceğiniz için dikkatli olmanız gereken bir işlemdir.

  1. Yeni bir dal oluşturmak için git checkout komutunu ve -b bayrağını kullanarak yeni dalın adını belirtin:

>>> git checkout -b <branch_name>                

EN ÇOK KULLANILAN 10 GIT KOMUTU

  • git merge

git merge, farklı dallardaki değişiklikleri birleştirmek için kullanılan bir Git komutudur.  Bu komut, bir daldaki değişiklikleri başka bir dala entegre etmenizi sağlar.

git merge komutu nasıl kullanılır?

  1. Terminali açın ve çalışma dizinine geçin. Çalışma dizini, Git deposunun bulunduğu dizini temsil eder.
  2. Birleştirmek istediğiniz hedef dala geçin:

>>> git checkout <branch>                

  1. Hedef dalın üzerine birleştirmek istediğiniz kaynak dalı belirtin ve git merge komutunu kullanın:

>>> git merge <branch>                

Git otomatik olarak değişiklikleri birleştirmeye çalışır ve çakışma durumunda sizden müdahale isteyebilir. Çakışmaları çözmek ve birleştirme işlemini tamamlamak için düzenleme yapmanız gerekebilir.

  1. Çakışma durumunda, Git size çakışan dosyaları gösterecektir. Dosyaları düzenleyerek veya çakışmayı çözerek değişiklikleri birleştirme işlemini tamamlayabilirsiniz. Daha sonra, değişiklikleri commitleyerek birleştirme işlemini kesinleştirebilirsiniz.

Bu şekilde, git merge komutunu kullanarak farklı dallardaki değişiklikleri birleştirebilirsiniz. Bu komut, eşzamanlı olarak farklı özellikleri veya düzeltmeleri geliştirirken ve farklı dallarda çalışırken proje yönetiminde yaygın olarak kullanılır.
